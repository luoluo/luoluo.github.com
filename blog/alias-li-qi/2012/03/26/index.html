Alias 利器

Mar 26th, 2012 | Comments 

适用场合：惯用指令很长
使用目的：简化指令，提高指令输入效率
使用规则：
    alias AliasName='NameToBeSimpler'   //注意：等号前后不允许空格。
    unalias AliasName='NameToBeSimpler'  //取消以上的别名设置 
举例：
    alias rg='rake generate'
    此时在命令行下输入
    rg 与 rake generate 作用一致

按照以上的设置在重新开机后，输入

lo@lo:/etc$ rg
rg：找不到命令

说明，rg命令已经失效了。为了使我们设置的别名长久有效，要把别名写在指定位置的文件里。

lo@lo: vim ~/.bashrc

此时使用rg，仍然是无效的，只有重启才可以生效。当然还有另外一种办法可以让他立即生效：

lo@lo:source ~/.bashrc

下面列一下我用的alias

alias rg='rake generate'
alias rd='rake deploy'
alias update='sudo apt-get update'
alias ..='cd ..' #据说是死罪
alias df='df -h'  #以M为单位查看磁盘空间
alias is='sudo apt-get install'

alias 是一把利器，但是最好不要使用的尺度太大，免得离开自己的机器就会失去“安全感”。小用宜情，乱用伤身。
